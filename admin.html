Wiki & Guidelines

Quick reference for managing all platforms that power LinguaBridge.

local_fire_department

Firebase Console

Auth · Firestore · Storage

open_in_newOpen
1

Manage user accounts

Authentication → Users tab. Reset passwords, disable/delete accounts, check last sign-in times.

2

View & edit database records

Firestore Database → collections: users languages tutors. Edit fields directly in the console.

3

Set a user role

Firestore → users → pick document → set role field to admin, tutor, or student.

4

Tutor photo storage

Storage → tutors/ folder. Photos are uploaded automatically when you save a tutor here. You can delete old files manually.

5

Security rules

Firestore → Rules tab. Current rules require auth. Only change if you need public read access for new collections.

article

Sanity Studio

Blog posts · Content management

open_in_newOpen
1

Create a new blog post

Studio → Content → Blog Posts → click the pencil/create icon. Fill in title, slug (auto-generated), cover image, excerpt, and body.

2

Writing content (Portable Text)

Use the body editor for headings, paragraphs, bullet lists, and images. Bold/italic inline marks are supported. Keep blocks clean for best translation results.

3

Categories & tags

Set the category field — it powers the filter tabs on the blog page. Use existing categories for consistency; new ones appear automatically.

4

Publishing

Click Publish in the top-right to make the post live. Drafts are saved automatically but won't appear on the website until published.

5

Translations

Blog content is auto-translated via Google Translate when the reader switches language. Write posts in English for best results. Session cache lasts the browser session.

school

Moodle Academy

Courses · Students · Enrollment

open_in_newOpen
1

Enable auto-login (one-time)

Site Admin → Mobile app → Enable mobile web services. This lets students enter Moodle from the portal without a second login.

2

Create a new course

Site Admin → Courses → Manage courses → Create new course. Set a short name, full name, and enrollment key if desired.

3

Enrol a student

Open the course → Participants → Enrol users. Search by username (the Moodle username stored in Firestore as moodle_user).

4

Student Moodle credentials

Each student in Firestore has moodle_user and moodle_pass fields. These are used for auto-login from their portal.

5

Web service token

The admin API token is stored in moodle-api.js. Rotate it in Site Admin → Server → Manage tokens if compromised.

folder_open

Website Files

Structure · Translations · Deployment

Static HTML
1

Key files

translations.json — all static UI text in 4 languages. language-manager.js — loads translations and handles language switching. translate.js — Google Translate helper for Sanity content.

2

Adding a static translation

Add a data-i18n="key.name" attribute to the HTML element, then add the key under EN/BG/RU/ZH in translations.json.

3

Prices & language data

prices.json is the current source for the Languages page. When Firestore integration is connected, this file will be superseded by data edited in this Admin portal.

4

Media files

Tutor photos live in Media/ folder locally and Firebase Storage for uploads via this portal. Logo, flags, and icons are loaded from CDN.

5

Login / role flow

login.html redirects by role: admin → admin.html, tutor → tutor-portal.html, student → student-portal.html.
